Git 학습 내용

버전관리 툴이다.

커밋은 메소드 단위마다 하도록 한다.

코딩, 커밋하는 루틴을 익힌다.

git 루틴

$git clone 주소
$git branch XXX
$git branch -a
$git switch XXX
$git status
$git add xxx
$git commit
with conventional commits
$git push origin main
($git push -u origin stem)
$git branch -D stem

git 수정

$git commit –amend
$git reset HEAD XXX
$git revert ~

conventional commits

prefix

feat: 기능 관련
fix: 오류 있을때 패치
docs: 문서작업
style: 코드 포매팅
conf: 환경 설정

git issue$project

Github flow 에 대해서

브랜치는 기능마다 하나씩 만든다.
마지막 최종 후에는 머지하지 않음.
깃허브가 알아서 함

git push -u origin XXX
remote 브랜치에 없는 상태일때만 upstream set

머지는 풀리퀘스트로된다.
로컬 메인에는 머지된 데이터가 없다.
풀해서 오리진 가져오고
작업 브랜치는 로컬에서 삭제한다.
리모트에서도 삭제한다.
이렇게깃플로우 한단위가 끝남

팀장, 팀원이 협업할때

한달짜리 프로젝트면
스프린트 1~2주 정도씩해서 돌려야 지치지 않음
스프린트마다 동작하는 결과물이 나올때마다 체크함

1. 팀장

new organization 생성, 팀원 추가, 오너는 추가 없음 팀장만
new repository 생성, public
환경설정 git clone, gitignore.io 로 gitingnore 생성
대상파일 하나 만들기 main.py
gitingnore, main.py 각각 add, commit
issue templet 생성, settings 에서 templet 선택, Sprint Backlog, 할일 추가시 사용할 것, 템플릿 내용대로 충실히 작성. 템플릿 커밋
환경설정 완료했음을 팀원에게 알림

2. 팀원

할일이 정해져있다면 이슈에 할일 작성, task 하나마다 커밋함
포크를 만들어서 로컬에 클론함 team repo 아님
브랜치 생성 후 브랜치에서 작업, 커밋, 테스크 체크
마지막 커밋시 resolve #1
브랜치에 푸시하면 한단위 작업 종료
팀장 컨펌 후
포크 레포에 있던 작업을 컴페어 앤 풀리퀘스트, 타이틀, description 잘 작성,

3. 팀장

어사이니 배정
코멘트, 컨펌, 리퀘스트 체인지 위해 코드 리뷰
팀원
풀리퀘스트 끝나기 전까지는 생성한 브랜치 남겨둠
수정요구사항 작업 후 요구사항마다 하나씩 애드, 커밋, 푸시,
팀장
작업 사항 이력 확인 후 리뷰 수정, approve
머지
팀원
이슈에 resolve
생성한 브랜치 제거

2~3 반복

팀주소로부터 git remote add upstream~~
git fetch upstream main
git merge FETCH_HEAD
conflict 발생시 해결

업스테이지에서 AI 부트캠프 Python 학습 정리

UpstageAi lab 2주차 이다.

4.3부터 4.9까지 김인섭 강사님과 Python을 배웠다.

Python은 이미 배웠던 언어라서 쉽게 배울 수 있었다.

하지만 이번엔 머신러닝, 딥러닝을 위한 Python을 배운다.

이전에는 Python을 배울 때 데이터 분석을 위한 Python을 배웠다면 이번엔 머신러닝, 딥러닝을 위한 Python을 배운다.

파이썬 기본 문법부터, 크롤링, FastAPI를 배웠다.

기본 문법은 복습 개념이였고, 크롤링을 제대로 진행해보았다.

크롤링으로 DBpia에서 논문을 엑셀로 옮겨오거나, yes24에서 책 정보를 수집하기도 했다.

과제로 플레이스토어 댓글을 수집하는 크롤링도 만들었다.

나는 ‘키우기’라는 검색어로 검색한 뒤 최상위 앱의 댓글을 수집했다.

댓글을 100개 수집해야해서 스크롤 명령을 내리는 자바스크립트가 필요했던 점 외에는 쉬웠다.

그리고 FastAPI를 학습했는데 이날은 조금 수강생들이 환경설정에 어려움을 겪어 집중이 잘 되지 않았다.

FastAPI는 처음 접해보는데, Flask와 비슷한 것 같다.

4.11인오늘 부터는 팀프로젝트를 진행하는데 트렌드 크롤링을 주제로 잡고 있다.

팀프로젝트를 잘 수행할 수 있도록 연습하는 과정이라고 보아야겠다.

업스테이지 AI 부트캠프 2일차 학습 블로그 시작?

Posted on 2024-04-25

UpstageAi lab 두번째 날이다.

오전에 입과 테스트를 보았다.

파이썬 기초와 머신러닝, 딥러닝 사전 지식을 묻는 1차 테스트가 있었다.

파이썬 기초 문제는 할만 했는데 머신러닝, 딥러닝 지식은 사전 테스트보다 쉬웠지만 이번엔 그냥 풀었더니 어려웠다.

어쨌든 채점하니 80점… 모르는 내용인데 문제를 보고 답이 될만한 것을 고르니 맞는 것도 있다.

이번 부트캠프를 통해서 이런 지식을 쌓아갈 수 있을까?

어느 분야든 마찬가지지만 AI분야는 공부하다보면 공부 속도보다 자료가 쌓이는 속도가 빨라서 매우 압도되곤 한다.

전반적인 AI 지식을 학습하기보다는 당장 내가 필요한 분야에서 지식을 찾는게 현실일 것 같다.

두번째 테스트는 여러 건물의 전력량 관련 데이터를 기반으로 머신러닝, 딥러닝을 이용하여 특정 기간 전력량을 예측하는 문제였다.

이번에는 GPT도움 없이 하다보니 이전에 특허DB로 워드 클라우드 만들때 썼던 pandas 정도 활용하는데 그쳤다.

실제 머신러닝, 딥러딩 관련 라이브러리는 검색해서 써보는게 아니면 기억하고 있지 않아 모르겠다고 적었다.

당연히 예측 결과도 도출하지 못했다.

그래도 만족한다…

현재 글을 작성하는 시간은 학습 블로그를 작성하는 시간이다.

블로그는 네이버 블로그랑 스팀잇을 운영해봤는데 github.io로 운영해보려고 한다.

그래서 지금 css도 없는 html 사이트 급하게 GPT에게 요청해서 이렇게 만들고 있다.

VScode로 글을 무식하게 작성하고 있으니까 글이 제대로 작성되고 있지 않다.

일일이 p태그를 넣어가며 글을 쓰고 있다.. 시간 나는대로 이 웹사이트는 완성하도록 하고 글은 콘텐츠가 중요하니까!

주제는 AI lab의 진행 각오인데 이번 년도의 큰 과제이니 즐겨보려고 한다.

또 하나의 Connecting Dot이 될 것이다

업스테이지에서 AI 부트캠프 1일차 오티 후기

UpstageAi lab 3기 오티날을 회고한다.

오티까지 각종 서류에 서명을 하고, hrd에서 수강신청도 하고, 슬랙 가입 등을 마쳤다.

과정에 대한 OT는 다른 자바 코스의 최종 결과물 제출로 바빠서 제대로 집중해서 듣지 못했는데 커리큘럼 소개가 대부분 이였다.

이 코스가 지향하는 바가 무엇인지 명확하지는 않았는데 커리큘럼 시트가 공유되어 확인하니 감을 잡을 수 있었다.

파이썬을 일주일만에 끝내고… 작은 프로젝트를 진행하는 식으로 몇주를 보내는데 오전부터 오후까지 일정이 꽉 잡혀있다는 인상이다.

그리고 김용담 강사님의 마인드 셋 특강도 인상 깊었다.

마찬가지로 AI lab출신으로 강사로 들어오셨는데 여러 수강생의 질문을 소상하게 받아주셨다.

최근 AI 분야의 트랜드, AI분야에서 아이폰 임팩트가 필요하다는 이슈, 영어의 중요성 등이 기억에 남는다.

2일차에는 오전부터 또 입과 테스트가 있다고 한다. 문제를 살짝 미리 보니 처음 사전 테스트와 비슷한데 큰일이라고 생각했다.
