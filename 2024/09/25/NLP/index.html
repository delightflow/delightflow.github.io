NLP 학습과 대회 후기

Posted on 2024-09-25

대회 진행 내용 히스토리

베이스라인 시도

  • 베이스라인: 41.7
  • 하이퍼파라미터 변경: topk ngram
  • 모델 변경: gogamza/kobart-summarization -> 41.8

LLM 실험

  • Claude haiku3 solar: GPT 4o mini 요약 프롬프트 -> 20~31점
  • GPT4o mini: 영문 번역 후 영문 요약 후 한글 번역 -> 33점
  • GPT4o mini 파인튜닝: 100개 dev 데이터로 진행 -> 40점

T5 모델 실험

  • 모델: lcw99/t5-large-korean-text-summary
  • Encoder max len: 1000, Decoder max len: 200
  • 결과: 43.92 (ngram, beam, max_len 등 변경 후 개선 없음)

LLM 추가 시도

  • GPT4o mini 파인튜닝: 100개 영문 번역 후 영문 요약 후 한글 번역 (한글 대화 원본 제공) -> 40점
  • temp 0.1: 42점
  • 200개 데이터로 파인튜닝: 점수 -> 43.93 (T5 모델 결과 약간 초과)

최종 결과

  • LLM GPT4o mini 파인튜닝: 100개 dev 데이터 모델 temp 0.1 시도 -> 44.4014
  • 결과: 너무 좋게 나옴
  • 한영 영한 번역 필요 없이 바로 진행: train 데이터 2000개로 파인튜닝 -> temp 0.1 -> 44.6 종료

시사점

  • T5 large 변경에도 불구하고 생성물에 차이가 거의 없었다는 것이 특징.
  • LLM이 입력과 출력 사이 요약 특징을 파인튜닝을 통해 학습한 것으로 파악됨.
  • temp를 낮추자 창의성이 줄어들고 학습된 스타일대로 출력하는 빈도가 증가하여 결과가 급격히 좋아짐.
  • 영한, 한영 모델의 경우 한글 대화 원본을 제공하여 번역 코퍼스로 활용하였고 점수 개선. 그러나 영한 번역된 대화를 영문 요약하는 과정에서 유실된 내용이 크게 존재함.
  • 파인튜닝의 학습 데이터 숫자는 OpenAI의 Docs에 따르면 100개 정도가 적합하나, 이번 과제는 학습 데이터 200, 400, 2000으로 늘릴수록 점수 향상.
  • T5 모델 학습은 20시간 소요, LLM 파인튜닝은 20분 미만.
  • GPT4o 모델을 사용하거나 파인튜닝 데이터를 선별해서 제공하는 추가 진행은 가능하나, 결과 차이는 비용, 시간 고려 시 불필요하다고 판단.
