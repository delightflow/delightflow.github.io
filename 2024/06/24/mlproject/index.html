캐글 프로젝트

첫 캐글 프로젝트를 진행했다.
미국 도시의 집값을 예측하는 문제로아래와 같은 변수를 가진 데이터셋을 이용하였다.

House_id : 주택별 아이디

Date : 주택 판매 날짜

Location : 주택 위치(도시 이름)

Type: 주택 유형(원룸, 투룸, 쓰리룸, etc)

Block: 블록(주소)

Street: 거리(주소)

Storey_range: 층 범위

Area_sqm: 면적(평방 미터)

Flat_model: 평면 모델

Commence_date: 판매 시작 날짜(준공일)

Price: 미래 주택 가격(타겟 변수)

https://www.kaggle.com/competitions/simple-housing-price-prediction/overview

이번 프로젝트를 하면서 느낀 점은 GPT-4o의 ML 성능이 캐글 초보자 수준은 넘으니 이보다 더 나아가려면 독창적인 해결법이 필요하다는 점이다.
6/3 : GPT로 개요 잡기 캐글 자료 다 넣어서 개요 짜고 약간 손보기 - 23,000-> 13,900
6/3 : 원 데이터셋에서 범주형 데이터 원핫인코딩 후 랜덤포레스트 -  13,400
6/4 : 원 데이터셋에서 EDA 결과 불필요 칼럼 제거 및 ‘type’ 타겟인코딩, 랜덤포레스트 - 13,900
6/5 : 원 데이터셋에 금리 데이터 추가 후 XGB, 랜덤포레스트와 앙상블 등 모델변경 - 14,000 
6/7 : 주택공급데이터 추가 후 모델 테스트 (ElasticNet, XGB, 앙상블 등)-15,900
6/8: 원 데이터셋에서 범주형 데이터 원핫 인코딩 후 XGBoost를 이용한 하이퍼파라미터 조정 - 13,400대 점수
6/10: 날짜 형식을 월, 년 분리에서 월 수로 변환하여 전처리 후 원핫 인코딩 - 13,100대 점수로 향상
6/11: 월 수로 년도 전처리 원핫 인코딩 후 금리 및 주택 공급 데이터 추가하여 다시 모델링 - 14,000
6/12 위도 경도로 지오코딩 후 재시도 - 점수 향상 없음
6/13: 위도 경도와 다운타운과의 거리 피처 엔지니어링 추가 - 13,900 
6/16: 위도 경도 기반 NearestNeighbor 주택 평균 가격 피처 엔지니어링 - 13,500
6/16: 위도 경도 기반 K-means Clustering 피처 엔지니어링 - 12,900 
6/17: 도시별로 데이터 분할 후 개별 모델 학습 및 테스트 - 12,700 최종 종료

맨 처음 개요는 모든 캐를 자료를 GPT에 입력해서 만들었는데조금 손보니 바로 39명 중 5등의 결과가 나왔다.

몇가지 돌파구를 찾기 위해 모델링, 피처 엔지니어링, 기타 기법을 사용했다.

날짜는 가격에 더 핏하게 맞출 수 있도록 월수로 변경하고, 지역 범주의 데이터를 강화한 점, 지오코딩을 시도해 본 점, 도시별로 데이터를 분할 후 각자 학습시킨 점 등이 주효 했다.

지오코딩 후 전처리하는 과정이 가장 오래걸렸는데 유효할 것이라고 생각했던 변수가 실제로 모델 성능 향상에 영향을 미치지 못했다.
다시 이 위도, 경도를 유효한 변수로 바꾸는 과정이 필요했는데 공들인다고 해서 좋은 결과를 보장하지 않는다는 점을 느꼈다.
