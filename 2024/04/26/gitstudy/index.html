Git 학습 내용

버전관리 툴이다.

커밋은 메소드 단위마다 하도록 한다.

코딩, 커밋하는 루틴을 익힌다.

git 루틴

$git clone 주소
$git branch XXX
$git branch -a
$git switch XXX
$git status
$git add xxx
$git commit
with conventional commits
$git push origin main
($git push -u origin stem)
$git branch -D stem

git 수정

$git commit –amend
$git reset HEAD XXX
$git revert ~

conventional commits

prefix

feat: 기능 관련
fix: 오류 있을때 패치
docs: 문서작업
style: 코드 포매팅
conf: 환경 설정

git issue$project

Github flow 에 대해서

브랜치는 기능마다 하나씩 만든다.
마지막 최종 후에는 머지하지 않음.
깃허브가 알아서 함

git push -u origin XXX
remote 브랜치에 없는 상태일때만 upstream set

머지는 풀리퀘스트로된다.
로컬 메인에는 머지된 데이터가 없다.
풀해서 오리진 가져오고
작업 브랜치는 로컬에서 삭제한다.
리모트에서도 삭제한다.
이렇게깃플로우 한단위가 끝남

팀장, 팀원이 협업할때

한달짜리 프로젝트면
스프린트 1~2주 정도씩해서 돌려야 지치지 않음
스프린트마다 동작하는 결과물이 나올때마다 체크함

1. 팀장

new organization 생성, 팀원 추가, 오너는 추가 없음 팀장만
new repository 생성, public
환경설정 git clone, gitignore.io 로 gitingnore 생성
대상파일 하나 만들기 main.py
gitingnore, main.py 각각 add, commit
issue templet 생성, settings 에서 templet 선택, Sprint Backlog, 할일 추가시 사용할 것, 템플릿 내용대로 충실히 작성. 템플릿 커밋
환경설정 완료했음을 팀원에게 알림

2. 팀원

할일이 정해져있다면 이슈에 할일 작성, task 하나마다 커밋함
포크를 만들어서 로컬에 클론함 team repo 아님
브랜치 생성 후 브랜치에서 작업, 커밋, 테스크 체크
마지막 커밋시 resolve #1
브랜치에 푸시하면 한단위 작업 종료
팀장 컨펌 후
포크 레포에 있던 작업을 컴페어 앤 풀리퀘스트, 타이틀, description 잘 작성,

3. 팀장

어사이니 배정
코멘트, 컨펌, 리퀘스트 체인지 위해 코드 리뷰
팀원
풀리퀘스트 끝나기 전까지는 생성한 브랜치 남겨둠
수정요구사항 작업 후 요구사항마다 하나씩 애드, 커밋, 푸시,
팀장
작업 사항 이력 확인 후 리뷰 수정, approve
머지
팀원
이슈에 resolve
생성한 브랜치 제거

2~3 반복

코드 가져오는 방법

팀주소로부터 git remote add upstream~~
git fetch upstream main
git merge FETCH_HEAD
conflict 발생시 해결
